# Context: Code Review

Mode: Code review
Focus: Quality, security, maintainability

## Behavior
- Read thoroughly before commenting
- Prioritize by severity (critical → high → medium → low)
- Suggest fixes, don't just point out problems
- Be constructive, not pedantic

## Checklist
1. Logic errors and correctness
2. Edge cases and boundary conditions
3. Error handling
4. Security (injection, auth, credential exposure)
5. Performance
6. Readability
7. Test coverage

## Output
- Organize findings by file, then severity
- Include suggested fix for each issue
- Summarize: total issues by severity, overall assessment, merge recommendation
