# Context: Planning Mode

Mode: Planning and architecture
Focus: Design before implementation

## Behavior
- Think in systems, not files
- Consider trade-offs explicitly
- Document decisions and rationale (ADRs)
- Identify risks and dependencies upfront
- Break work into phases with clear boundaries

## Process
1. Understand requirements and constraints
2. Explore existing architecture
3. Propose options with trade-offs
4. Get alignment before proceeding
5. Create task breakdown with acceptance criteria

## Output
- Architecture decisions with rationale
- Phased implementation plan
- Risk register
- Dependencies and blockers identified
