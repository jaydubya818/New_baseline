# Second Brain — AI Navigation

> Read this file first. It tells you how to navigate and update this vault.

## System

This is my second brain — a structured knowledge base of everything about me, my work, and my business. Use it to answer questions with full context and to save new information as I work.

## File Structure

| Folder | Purpose | When to Read |
|--------|---------|-------------|
| `context/` | Who I am, business, strategy, ICP, brand, team | Always — start here for background |
| `daily/` | Daily session logs with summaries and decisions | When resuming work or checking history |
| `intelligence/` | Meeting transcripts, decisions, market research | When asked about past discussions or decisions |
| `projects/` | Active project context, status, notes | When working on a specific project |
| `resources/` | Reusable prompts, templates, frameworks, examples | When creating content or following a process |
| `skills/` | Reference files that skills point to (ICP, voice, etc.) | When executing a skill |
| `tasks/` | Current to-do lists and task tracking | When asked what to work on |

## Knowledge Routing

- Questions about me or my business → `context/`
- Questions about past meetings or decisions → `intelligence/`
- Questions about what to focus on → `tasks/` + `daily/`
- Questions about a specific project → `projects/{name}/`
- Writing content → `skills/` for reference + `context/` for ICP/voice
- General how-to or frameworks → `resources/`

## Rules

1. **Save when asked** — When I say "remember this" or "save this", write to the appropriate file
2. **Daily logs** — Log session summaries to `daily/YYYY-MM-DD.md` (what was done, decisions, next steps)
3. **Wiki links** — Use `[[double brackets]]` when referencing other documents so Obsidian tracks connections
4. **Follow links** — If a document contains `[[wiki links]]`, follow them to find related context
5. **Create files as needed** — If a topic doesn't have a file yet, create one in the right folder
6. **Update, don't duplicate** — Check if information already exists before creating new files

## Updating Context

When I make a decision or correction in conversation:
1. Identify the right file (e.g., writing preference → `context/writing-preferences.md`)
2. Append or update the relevant section
3. Confirm what was saved and where

## Session Start Checklist

When starting a new session:
1. Read this file (you're doing it now)
2. Check `tasks/current.md` for active work
3. Check `daily/` for the most recent log
4. Pull additional context as needed based on the task
